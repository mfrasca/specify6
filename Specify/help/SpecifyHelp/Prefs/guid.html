GUID

GUID Overview

A field for a unique identifier exists within the Collection Object, Taxon, Geography, Lithostratigraphy, Locality, Agent, Reference Work and Journal tables in Specify. The field is named Globally Unique Identifier, (GUID).

A more specific form of GUID, Life Science Identifiers (GUIDs) have become the universally accepted identification scheme for the biological collections community. Using an GUID to identify information allows that information to be shared and recognized globally. As more biological information systems use GUIDs it will become possible to instantly recognize when globally exchanged information has been updated or duplicated.

Specify has adapted its own scheme for creating GUIDs, but was also written to include the flexibility for institutions wanting to include their own scheme to create a plugin.

Specify’s GUID scheme was created with specific sections so that information about the referenced information is instantly recognizable. This scheme includes a version number, which will change each time the record is updated.

 

GUID Authority refers to the source of the data, which is typically the url for the institution. This is edited using the Institution form (System > System Setup > Configuration).
Institution Code is the code located within Specify for the Institution. This is edited using the Institution form (System > System Setup > Configuration).
Collection Code is the code for the collection. This is edited using the Collection form (System > System Setup > Configuration).
Collecton Code refers to the type of record. This is edited using the Collection form (System > System Setup > Configuration)
Catalog Number is the catalog number for the data in Specify.
Version refers to the amount of times the information has been updated.

The Catalog Number is used within the GUID schema to identify specimen records, but all other records are identified using the primary record id found within MySQL.

GUIDs can be automatically created by Specify for each specimen, taxonomy, geography, lithostratigraphy, locality, person, reference work and journal record located within Specify. GUIDs are configured in the System Preferences.

 

GUID Preference

For Mac: Choose Specify > Preferences > GUID.
For Windows and Linux: Choose Edit > Preferences > GUID.

Note: The GUID preference is only available to the Administrator by default in Specify. If GUID does not appear in your system preferences tool bar please visit your Specify Administrator to have your permissions changed.

GUID Preference

  • Automatic GUID Generation
    Check the box to choose tables for automatically create GUIDs. This will allow new records in each specific table to include an automatically generated GUID. To automatically generate GUIDs for existing records in tables first choose the tables in this section, then click the Create GUIDs button.
  • Additional Options
    Click the Create GUIDs button to automatically generate GUIDs for existing records in the tables selected in the Automatic GUID Generation section.

Specify will not override GUID information.
