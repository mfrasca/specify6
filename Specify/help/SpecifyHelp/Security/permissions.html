Adding Users and Setting Permissions

Specify includes a highly flexible Security module that allows a system administrator to add users, then limit their operations within and access to the various modules and components in Specify. These are essentially the User Permissions.

To access the Security tool click System > Collection Setup > Security Tools.

 

Users are added to Specify based on their Institution Level and User Group.

Institution Levels

Specify mirrors the common levels of heirarchy within institutions. This hierarchy is viewed in an Institution Tree on the left side of the Security window.

  • Institution
    • Division
      • Discipline
        • Collection
          • Users

At this time users are added at the Collection and Discipline levels. The lowest level, Collection, can access data within a specific Collection (Wet Collection, Teaching Collection). Specify opens in a collection, or at a Collection level; therefore users added to only one Collection would not have the ability to access other collections. Collections themselves are held within a Discipline (Ichthyology). Users added at the Discipline level can access all Collections within a Discipline.

Users can belong to more than one Instituion Level.

 

Adding Institution Levels

Institution levels are not added in the Security Tool, they are add at System > Collection Setup > Configuration.

 

Editing Institution Levels

To edit a specific user, collection, discipline or division simply click on it. A form will open in the Workspace for editing data.

 

User Groups

Specify adds users to an existing user group. The groups themselves were established according to their access to data and tools within Specify and have a default set of priviledges. Users entered into the group acquire the default priviledges. In a later version of Specify the default priviledges will have the ability to be edited per the individual user as well as per user group.

  • Guest users can view the data within a Collection or Discipline only.
  • Limited Access can view and edit most tables but can not access all the Specify modules such as trees.
  • Full Access users require access the tools for inputing and editing data as well as most modules within Specify, but may access the security module for adding users and setting permissions.
  • Manager can add users and give permissions as well as access and edit all data and modules.

Users can be added to more than one group within different Disciplines, but should not be added to both a Discipline and Collection within the same Discipline.

 

Adding a User

Right-click on the appropriate Group type within the applicable Institution level to add a user to a group. A dialog will appear to either add a new user or existing user.

Existing User

This is the beauty of the implementing the Institution Heirarchy at the Security level; users can be given unique permissions to different collections without needing more than one password.

New User

Users are added in the Specify User dialog.

  • Name refers to the username the user will use to login to Specify. Once the username is saved it may not be edited.
  • Password is the password the user will use to login to Specify. Passwords can be edited at a later date either in Security Tools or by choosing Help > Change Password.
  • Email
  • Agent refers to the data in the Agent table. All users must be Agents. If the user already exists within the table type the first few letters in the name and either type the down-arrow key.
  • Key refers to the encrypted Master Username and Password. Simply fill in all the information within the dialog and click the Generate New Key button. The key can not be created without the Name and Password information. The key is stored locally and will therefore need to be given to the user and entered by the user at the time of their first login to Specify. For more information regarding this key please refer to Security.

 

Permissions

Specify 6.0 grants default permissions to each User Group within Specify. In future releases group permissions will be set individually.

 

Permissions are divided into categories, with each category listing its various components in a separate pane:

  • Data Objects refer to the Form system inside Specify that allows data to be viewed, added, edited and deleted.
  • Tasks refer to all other tools within Specify. These tools do not affect actual data, but allow users to manipulate the data in meaninful ways (Trees, Queries).
  • Preferences lists the modules in the actual Preferences module.

Each of these categories is further distinguished by user capabilities, or actions:

  • View allows a user to view data within the component.
  • Add allows a user to add data or create a new item.
  • Modify allows users to edit data within an item.
  • Delete allows users to delete data within the component.

 

Permissions and how it affects the Specify Application

When Specify starts up it checks the user permissions and loads various components accordingly. Therefore, users that are not given permission to use modules in Specify will not see the corresponding buttons that connect users to that module. For instance, users that are not given permission to edit trees will not have a tree button on the task bar or have the ability to add the tree button in the user preferences tool.

Similarly, users that are not given permission to delete or add data will not see the add or delete buttons.
