Schema Configuration

The Schema Configuration module is a highly functional, though somewhat complex set of tools created to configure Specify for individual Discipline needs. Changes made in the Schema Configuration will affect all Collections in the Discipline. For this reason, each Discipline is encouraged to tailor Specify to their needs as part of the Specify Setup directions.

The tools in Schema Configuration include:

Hide unused Tables or Fields

Change the label of Tables and Fields in Specify. The names in the schema are not editable, but the display names, referred to as labels can be edited.

Read/write descriptions (usage notes) for Tables and Fields.

Format a Table

Aggregate a Table

Format a Field as a Pick List, Weblink or Numbering Format.

Format a Field as a Weblink

Specify 6 was created to handle all disciplines. The Schema Configuration tool allows users to hide tables and/or fields that are not being used for a particular discipline. This not only customizes Specify 6 for a specific discipline but also enables the program to run more efficiently.

The Schema Configuration tool can be used to:

hide tables and/or fields not being used in an individual discipline

label fields ***will this be shown on forms then?***

write descriptions (usage notes)

associate a field to a pick list

format a text field

** Check to make sure that this is changing by Discipline and not just Collection!!!

 

Note: The Schema Configuration Tool is NOT a local tool, any changes made to the Schema will change every Collection throughout the Discipline.

Table choices

Table Preference Result Notes
Name Changes the name of the Table as represented in the Form System and Query Builder. Changes to the Table Name will be reflected on the Forms during the current session, but Specify must be relaunchedfor the changes to be reflected in the Sidebar lists.
Hide Table Hides the Table in the Schema. This will not remove the Table from the Form System. The Table will be removed from the Query Builder and Simple Search.

Hidden Tables that are used in the Form System will cause a Warning Dialog to appear when using the Form System. It is recommeded that Tables be removed from the Form System before being removed from the Schema.

Tables that are still used in the Form System but removed from the Schema are still editable and any changes made to the Table will be reflected in the Database.

Desc Description of the Table.  
***Spell Check Checks the spelling of words used in the Description.  
Display Format Launches the Display Formatting Editor (explained below). Tables with Many-to-One relationships are formatted to be represented by at least one field in both the Form System and Query Builder. Most tables are represented by only one field, (Collection Object shows the Catalog Number), but some are best represented by more than one field (Agent may be formatted to display last name, first name, middle initial).
Aggregate Opens the Aggregation Editor (explained below). Tables with a one-to-many relationship return aggregated data in the Query Builder. For example, a Collection Object can have many Preparations. The results of a query of Collection Objects wtih Preparations will return Preparations aggregated together in one column, separated by semi colons.
Web Links Opens the Web Link Editor (explained below).
