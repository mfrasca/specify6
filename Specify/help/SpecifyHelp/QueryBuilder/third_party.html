Exporting Data for a Web Portal/Third Party

The 6.5 release of Specify facilitates the creation of a web portal by using the Apache Solr search platform. Specify data can also be shared with third parties, such as Darwin Core or OBIS. Both instances of data sharing require two steps to create an index file. First, the schema of the third party must be mapped to the Specify schema using the Schema Mapping Editor inside Specify. Next, the data file is created and/or updated using the Data Exporter stand-alone application (found in the bin directory within the Specify application directory).

Creating a Web Portal using exported Specify data and Solr

Specify 6.5 harnesses the search capabilities of Apache Solr to automate the creation of a web data portal. Solr is a "popular, blazing fast open source enterprise search platform from the Apache LuceneTM project. Its major features include powerful full-text search, hit highlighting, faceted search, near real-time indexing, dynamic clustering, database integration, rich document (e.g., Word, PDF) handling, and geospatial search."

Creating a web portal using a Specify data and Solr requires several steps:

  1. Apache Solr and a server must be installed on the computer that will be serving data to the internet.
  2. Create and Export a Schema Mapping file from within Specify.
  3. Use the Data Exporter, a separate Specify tool, to create (build) an index of the data and other configuration files.
  4. Integrate the exported web portal files with Solr

Each of these steps is explained in a web portal document accessible on the Specify Project website: specifysoftware.org

Publishing Specify data through IPT

Specify 6.1 takes advantage of the Global Biodiversity Information Facility's (GBIF) Integrated Publishing Toolkit (IPT) to publish Specify data to the web. IPT is an open-source, Java web application designed to publish biodiversity data on the Internet. The IPT creates an HTML web site on your web server and starts a network service that publishes your collection's data with the TAPIR data communications protocol. GBIF can then be notified of the existence of your IPT server to make first contact and integrate your data into the GBIF cache.  You will be assimilated. Specify no longer uses its own web interface software nor the DiGIR protocol for serving XML encoded data. See http://ipt.gbif.org for additional information regarding IPT.

Using the Integrated Publishing Toolkit to display and publish, Specify collection data requires several steps.

  1. Tomcat must be installed on the computer that will be serving data to the internet. 
  2. Specify data must be mapped to schema authority, such as Darwin Core, from inside Specify.
  3. A separate Specify tool, Data Exporter, is then used to create (build) the data file for IPT.
  4. IPT must be setup to display Specify data.

Each of these steps is explained in the pdf document, Publishing Specify Data with IPT, found on the Specify website documentation page at http://specifysoftware.org/content/documentation.


The current document explains how to define, or 'map', the Specify fields for data export. Please refer to the Data Exporter page for documentation about generating the export data file.

Mapping Specify fields

Open the Schema Mapping Editor by choosing System > Schema Mapper. 

Two actions are available in the side bar:

  • Create New Mapping - either choose to create a new mapping by changing an existing supported schema (recommended when exporting data for third parties such as IPT) or by creating it (recommended when exporting data for a web portal).

Note:it is not necessary to include attachment fields in mappings, attachments are automatically included in mapping files.

  • Import Schema - when using a supported schema (example: Darwin Core) the schema file must first be imported into Specify, this will allow the schema to be added to the 'Create New Mapping' dialog.

Edit a supported schema

Click on Create New Mapping to open a dialog listing IPT supported schema files. Choose one from the list or import a new schema by clicking Import Schema and choosing one from your system file. Once a schema is chosen, a dialog will appear to enter a title and version. The two entries will be joined, or concatenated, together and used to identify the schema mapping in the external tool. The concatenation will also be used for the name of the database table which holds the cache.

Specify ships with the following schema files preloaded in the Specify/Config directory:

  • TDWG_DW_Core.xsd - Darwin Core 1.4 - draft standard under discussion (used in GBIF, and recommended to use in GBIF)

If a schema other than Darwin Core is desired, it must first be downloaded (preferably into the same Config directory) and imported as an .xsd definition file.  Three examples are:

OBIS - http://www.iobis.org/obis/obis.xsd
PaleoPortal - http://data.paleoportal.org/schema/conceptual/1.0/paleoportal.xsd
ABCD - http://rs.tdwg.org/abcd/2.06/ABCD_2.06.xsd

Note: Currently all schema mappings are assumed to be specimen-based, i.e: the elements described in the schema definition are assumed to refer to biological specimens. This means the mapping is based on a CollectionObject query by default.

To delete a schema from the available list (and your Specify/Config directory) right-click in the side bar (in an empty space), then choose Delete Schema in the resulting context menu. You may also delete the file directly from your Specify/Config directory.

Create a new schema

Click on Create New Mapping. The resulting dialog lists both IPT supported schemas and the term 'no schema'. To create a new schema (typically for exporting to a web portal) choose 'no schema'.

Once a schema has been chosen, a modified query builder window will appear.

Creating/Editing a Schema Mapping

The mapping tool looks and works similar to the Query Builder tool with the Schema Concept added. The Schema Concept refers to the name of the field that will be associated with the data exported from Specify.



When creating a schema map for exporting data to a web portal the name of the Schema Concept can be renamed.



When using a schema authority to share data with a third party the Schema Concept will display a pick list of all choices for fields in the associated schema.

Un-mapping a Term

To un-map a term, click the  (remove field) button at the right of the mapping element.  The field will not be removed from the query but simply un-mapped. 

 

Adding Un-mapped Conditions

IPT and Specify require that all uploaded records be unique.  Due to the one-to-many relationships contained in Specify databases it may be necessary to add certain conditional fields to the schema to ensure this e.g. adding the Determination Current field and setting it to Yes will eliminate multiple determinations.  One can also select a table to take advantage of the formatted and aggregated value for that table e.g. selecting the Collectors table will add all collectors on a single line.  Fields can also be added to eliminate certain records from the resulting cache

  • Select the 'Add Unmapped Condition' term located at the bottom of the list of schema terms. Then either double-click on a Specify field (or select the field and click the (add) button) to add it to the schema.

For additional help creating/editing a mapping schema please refer to the Query Builder help page.

Checking a Schema Mapping

  • Click the Preview button in the lower right corner to display a 120 record sample of the mapping in a query results display. If your data contains duplicate records, you will be alerted and will need to correct this problem by adding conditions to the query before continuing.  Once successful, Specify will display a dialog message asking you to use the external tool to export the data.
  • If all the mappings are correct, save the schema mapping by clicking the Save button.

The mapping name is automatically created by Specify, based on the title and version of the schema.

  • To edit a schema mapping simply click on it in the side bar to re-open it in the Schema Mapping Editor. 

Note: Whenever a schema mapping is edited, a full rebuild of its cache (in the separate Schema Exporter tool) will be required.

The mapping file will be saved as an xml file.

 

Import/Export a Mapping

Import

  1. Right-click in the side bar (in an empty space), then choose Import Mappings in the resulting context menu.
  2. Choose the file to import from the resulting file menu dialog

Note: When importing mappings that have the same name as existing queries, Specify will simply add a number (2) to the import query name.

 

Export

  1. Right-click in the side bar (in an empty space), then choose Export Mappings in the resulting context menu.
  2. Choose the mapping to export from the resulting dialog list and click the OK button.
  3. Name the file and choose a storage location for the file in the resulting file menu dialog, then click the Save button.

 

Delete a Mapping

Right-click on the side bar (in an empty space) then choose Delete Schema or drag and drop a mapping onto the Trash Can in the side bar to delete it

Note: Mappings that are deleted from within Specify will no longer be available for data file updates in the Data Exporter tool.

 

To generate the Data file see the documentation page Data Exporter.
