Column Mapping

Column mapping is creating an association from a column in the input file to a data member in the database. An association ensures that each column of each row of data in the input file has a place in the database. The proper database destination is a combination of a Data Object and a field.

Column Mapping Dialog

There are two ways these association are created: automatically and manually.

Manual Mapping

OnRamp will make an attempt at discovering a mapping for a column, but depending on what names you used as column headers the auto mapper may not be able to detmerine the desired mapping (See Auto Mapping). At this point you will need to manually choose a mapping.

In order for there to be a mapping the you must select three items.

Auto Mapping

Auto Mapping is done during the import process to see if the mapping from the data files to the database fields can be automatically determined. During the initial steps of importing a file you indicated whether the first row of data is the column headers. Data files with column headers are require less work to import. (See Importing Data Files) The data file is inspected during the import to determine what the each column's header might be.
