Data Set Overview

Data is entered, edited, organized and stored in a Data Set.

Data Sets are created with different Actions depending on the type of data you have.

To enter data from field notes or labels choose New Data Set to create an empty Data Set. Column headings for the new Data Set are chosen from fields within Specify 6, which ensures that any data entered into the WorkBench will import correctly into Specify 6. The column headings can be renamed, but will still maintain a link to the original Specify field. These links are referred to as 'maps'.

Data that has already been entered into either an MS Excel or CSV file can be imported into a Data Set. The WorkBench not only imports the data, but also helps map the existing column headings to fields in Specify 6 with an 'auto-mapping' feature that reads the import column headings and automatically maps them to any matching fields. Any remaining unmapped column headings can be manually mapped. The column headings can be mapped to different Specify fields later by editing the Data Set Mapping.

Data Sets display the imported column headings rather than the field names, as these are more familiar.

When data exists as a group of scanned card images or field notes, the WorkBench allows the user to Import Images, and connect these to individual records in a new Data Set.

The WorkBench can also link images individually to each row within an existing Data Set.

Data Sets are edited as either a Grid, or a Form. Each view offers unique functionality:


Grid Editing

Search Panel

Georefrencing with BioGiomancer

Mapping with Google Earth

Georeference Conversion tool

Export selected records
Form Editing

Rename Columns

Rearrange form layout of the Workspace

An entire Data Set, or selected rows can be exported to an MS Excel spreadsheet using the Export Data Set action.

A Data Set can be described, as well as named using the Data Set Properties dialog.
