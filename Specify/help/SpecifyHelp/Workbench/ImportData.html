Creating a Template and DataSet by Importing Data

OnRamp can easily import data from MS-Excel® or CSV files and convert it to a 'normalized' stateby reading each column in the import file, or column, and 'mapping' it to an appropriate field in the schema. These maps are saved as Templates and can be changed at any time. In fact, fields within a column remain de-normalized and can be renamed, but schema fields being 'normalized' can not. To rename a column field see 'Form Editing'.

To start importing data and mapping fields, click on the ‘Import Data’ button on the side bar. A File Dialog will appear enabling an import file to be chosen.

Using Column Mapper

The Column Mapper window is divided into three sections: Mappings, Data Objects and Fields.

Mappings refers to the relationship between the spreadsheet fields and the DataSet fields and tells OnRamp how values will be interchanged between the two files. More specifically, the field name on the left, Import, side of the window will supply values for the right, Database, side of the window. If a Database field shows 'no mapping' then no values will be transferred to the DataSet unless a relationship is created.

Data Objects lists the type of data that is available.

Fields are the names of the fields available for each Data Object.

To view available fields for the DataSet first select a Data Object.

Editing the Mappings

New fields can be added to the map or changed in the DataSet by using the following buttons:

adds a field name to both the DataSet and Data
Deletes the field name in both the DataSet and File
< Remaps the existing DataSet field name to the selected File field name by replacing the existing File field name
> Deletes the File field name and remaps the DataSet field name to ‘unmapped’. This will result in blank cells within the DataSet

To add a new field first choose the new Field name from within a Data Object, then use the (+) button at the bottom of the window. The new field name will be added to both the columns and schema.

To change a particular schema Field that is already mapped to a Column field first select the Column field name to be changed, then choose the correct Field name from a Data Object and use the < icon to move the field name.

When the desired fields in the File are correctly mapped to the DataSet Fields select the ‘ok’ button in the bottom right of the window. The new Template and DataSet will appear in the workbench window. It will automatically be named the same name as the File that was imported.

Quick Tip for Creating a Template and DataSet by Importing a File

    1. Choose ‘Import Data’ from the Sidebar
    2. Select the file name from the pop up window
    3. Answer the questions about the file in the next pop up window(s)
    4. Carefully review the map shown in the ‘mappings ’ window
    5. Add schema and import fields by choosing fields within the Data Objects provided and selecting (+)
    6. Change mapped import fields by selecting new fields within the Data Objects provided and clicking <
    7.  Change mapped DataSet fields to ‘no mapping’ by selecting the mapped DataSet Field and clicking >
    8. Delete mapped DataSet and Import fields by selecting them and clicking (-)
