About the Trees

Specify 6 stores various table data in heirarchical tree format. These include Taxon, Geography, Storage, Chronostratigraphy and Lithostratigraphy. These tables are also included in the form system, but the tree offers a more broad, visual glimpse of the data. Data can be edited and entered using both the form and tree system.

Each of these trees can be built 3 ways:

  • Manually by adding adding one record at a time.

  • Dynamically from an existinguser database. This may include a conversion from Specify 5, which the Specify team can help with, or imported through the WorkBench.

  • Dynamically by downloading and importing an authority file from an authority source.

Specify Tree Terminology

The various trees show relationships in a hierarchical structure. All hierarchies are referred to as ranks. All nodes immediately under a rank are referred to as children. Any node with a child is referred to as a parent.

Ranks are shown in columns on the tree. Tree column colors can be set in Preferences > Tree. See also Preferences

 

Using Trees

Open a tree by choosing Data Entry, in the tool bar, and clicking on the tree name in the Sidebar.

Tree ranks are defined using the Tree Definition tool; System > Collection Setup > (table name) Tree > Tree Definition. Before adding nodes to an empty tree, the Tree Definition must be completed.

Only one user may edit a tree, tree definition or the related data entry form at the same time. Simultaneous editing of a tree, tree definition or data form is also not permitted by a single user. Trees and forms will still be available for viewing, searching or querying. In order for a second user to use the tree or form, the first user must close the affected tab.

When a tree definition is open and the same user opens a tree the tree definition does not need to be closed, the tree will display in the tree sub pane.

Note: In the event that a tree becomes locked and the user of that tree can not be located, (which may occur due to improper shutdowns of the application), trees can be unlocked in System > Configuration > (table name) Tree > Unlock Tree Editing.

Dragging and Dropping

Nodes can be dragged and dropped within the tree module to move or synonymize nodes. When the screen is split, nodes can be dragged and dropped within the same screen or into the opposite screen. A highlight box will appear around any nodes that can accept the node being dragged. Once the node is dropped onto the accepting node a dialog will appear allowing the user to:

Reparent a node (and its children) to a new parent. Enforced ranks may not be skipped. This option is only available when dragging BETWEEN ranks.

Synonymize the node. To undo a synonymy right-click on the node and choose Undo Synonymy in the context menu. Synonyms are differentiated from non-synonyms by their text color. Synonym text color can be set in Preferences. Synonyms are also shows as a tool tip on the node. This option is only available when dragging WITHIN a rank.

Note: Synonymys created in the Taxon tree are captured in the Active Taxon field on the Determinations form.

Select Cancel on the dialog to place the node back in its original position.

 

Collection Object Counts

Counts of collection objects associated with each node can be enabled for all trees. This feature requires data processing and therefore is not enabled by default. Enable Collection Object Counts on a tree by choosing Preferences > Tree. See also Preferences

Counts are displayed next to the Node as two numbers (X,Y):

  • X represents the number of Collection Objects for the Node only.
  • Y represents the number of Collection Objects for the Rank and all nodes beneath it.
