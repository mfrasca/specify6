Configure Collection Relationships

Choose System > Configure Collection Relationships to open the tool.

 

Collection Relationships

 

This tool allows two collections to create a relationship in which there is a source collection and a destination collection. The source collection must always include collection objects. The source collection object forms will include a field for choosing the destination information. This may be either another collection object from an existing collection or taxonomic name from a taxonomic tree.

A single source may relate to either a single destination or many destinations within a collection.

The Destination collection object form does not offer the capacity for choosing a collection object or taxonomic name from the source collection.

Types of Relationships:

  • Object Relationship
  • Host Taxon ID

Object Relationship refers to two collections with collection objects in which a source collection object relates to one or many destination collection objects.

Host Taxon ID refers to the field name in which a source collection object relates to a destination taxonomic name. The taxonomic name is taken from a destination collection which exists merely for the purpose of offering a taxonomic tree. It is not necessary for the destination collection to include collection objects.

 

Source Forms

The source collection object form must be edited to include the necessary fields to show these relationships.

One Source to Many Destinations

 

Simply click on the (Search) button and type in a catalog number from the destination collection. Specify will search the destination collection and show results in the dialog. Choose the correct value from the list to add it to the relationship.

Select the (Remove) button to delete a destination value from the list.
