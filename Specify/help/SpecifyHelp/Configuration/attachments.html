Attachments

Many tables in Specify 6 allow Attachments. Specify copies any attached files to a specified Attachment Repository using a designated path. It is necessary to create the storage location before users will be allowed to create attachments. It is recommended that the attachment repository be on a shared network drive that all users have been given Read/Write/Delete permission for, rather than on a personal (local) computer. If attachments are stored locally they can not be shared.

Once the attachment repository has been designated it is necessary to communicate the path to the attachment repository. Users will then need to enter this Attachment path in their user Preferences.

  • For Mac: Specify > Preferences > System > Attachment Storage Location.
  • For PC and Linux: Choose Edit > Preferences > System > Attachment Storage Location.

Note: if the attached file is an image, Specify also creates and stores an icon for the image in the Attachment Storage Location.

If a path does not exist or Specify is not able to connect to the path the Attachment button will become disabled on the forms.

Accepted Attachment files include:

  • text
  • audio
  • binary
  • pdf
  • image (jpg, png, gif)
  • html
  • spreadsheet
