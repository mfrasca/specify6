Plugins

Specify 6 interfaces with the Google Earth, BioGeomancer and GEOLocate web services. Choose System > Plugins.

These web services can be launched by

  • Clicking the web service icon in the Sidebar and choosing a Catalog Number or Record Set.

  • Dragging and dropping a Collection Object, Locality or Collecting Event Record Set on the web service icon from within the sidebar. These Records Sets must include the Locality data listed below for each service.

Plugins that can accept a Record Set display a colored outline as the Record Set is dragged within the sidebar. An arrow will display within the outline when the Record Set can be dropped onto the plugin.

Specify is not responsible for any of the results of these web services, Specify is integrated with these services only.

 

Google Earth

Specify plots locality records (or associated Collection Objects or Collecting Events) in Google Earth based on latitude and longitude data provided in the locality form. All associated data is stored in a KML file which is launched within Google Earth upon clicking the Google Earth icon on the sidebar, or dropping a Record Set on Google Earth from within the Plugins Sidebar.

Collection Object information including the Collector, Catalog Number and Taxonomy is shown within the Google Earth information balloon. Specify also allows the insertion of two web service links for genus and species within the information balloon by configuring Google Earth at Specify > Preferences > Tools.

Google Earth can also be launched from within a Locality form by clicking the (Google Earth) icon.

Locality data Requirements

  • Longitude and Latitude

 

Google Earth must be installed on the local machine for the Specify Google Earth integration to work.

 

BioGeomancer and GEOLocate

BioGeomancer and GEOLocate both use a description of a Locality, and at least three geographic locations to find Latitude and Longitude data. This is refered to as Geo-Referencing.

Locality data Requirements

  • Localities; including county, state and country
  • Locality Name

A weblink icon is available on the Locality form for a quick link to one of these services. Choosing Specify > Preferences > System for the Mac or Edit > Preferences > System for Windows and Linux to select a service.

Once the required information is entered in the Locality form the icon for the weblink (at the bottom of the locality form) becomes enabled. Click the icon to launch the service. The geo-referencing data will display in a dialog which includes the locality information, count of results, longitude, latitude, and a map of the location. If the web service results represent more than one record each record can be stepped through by using the buttons at the bottom of the dialog:

Button Result
Help Launches Specify Help.
Accept

Accepts the locality data from the web service and copies it into the latitude and longitude fields. Specify also writes the name of the web service in the src lat long unit field, then steps to the next record.
Skip Skips the current record information and steps to the next record.
Quit Closes the dialog.

If the web service data is accepted and latitude and longitude information already exists in the locality form the existing information, including the source field, will be overwritten by the web service data.

Specify does not keep any other data that may be returned by these web services.
