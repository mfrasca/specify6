Security Tool

Specify includes a highly flexible Security tool that allows a system administrator to add users, then limit their operations within and access to each of the components in Specify at Institution levels. These are essentially the User Permissions.

To access the Security tool click System > Collection Setup > Security Tools.

 

Users are added to Specify based on their Institution Level and Group.

Institution Levels

Specify mirrors the common levels of heirarchy within Institutions in its user hierarchy. This hierarchy is viewed in an Institution Tree on the left side of the Security window.

  • Person
  • Group
  • Collection
  • Discipline
  • Division
  • Institution

At this time users are added at the Collection and Discipline levels. The lowest level, Collection, only has access to data within a specific Collection (Wet Collection, Teaching Collection). Specify opens at a Collection level; therefore users added to a specific Collection can never access other collections. Collections themselves are added within a Discipline (Ichthyology). Users added at the Discipline level can access all Collections within a Discipline.

Users can belong to more than one Instituion Level.

Note: Collection Object data is limited to the Collection level, but many related tables are available at Discipline and Division levels. This assurs that data is not unnecessarily repeated allows the data to be used as an authority file within an institution (for example, the Agent and Treatment Event table are available at the Division level, making the data available to all Collections and Disciplines within the Division.)

 

Adding Institution Levels

Institution levels can be added by right-clicking on the parent level and selecting the 'Add x' button from the Context Menu. For example; right-click on Discipline and click the Add New Collection button. A form to add the level will open in the Workspace.

Editing Institution Levels

To edit a name in the Institution Tree simply click on it. A form will open in the Workspace.

 

User Groups

Specify groups users according to their needs:

  • Guest users need to View the data within a Collection or Discipline.
  • Data Entry users need access the tools for inputing and editing data, but do not need to configure any of the tools.
  • Discipline Managers need full access to all data tools including the ability to configure the tools. Discipline Managers may or may not have access to System Tools within Specify.

Users can be added to more than one group within different Disciplines, but should not be added to both a Discipline and Collection within the Discipline.

 

Add a User

Right-click on the appropriate Group type within the applicable Institution level to add a user to a group. A dialog will appear to either add a new user or existing user.

Permissions

Specify 6.0 grants default permissions to each User Group within Specify. In future releases user permissions will be set individually.

 

Permissions are divided into categories, with each category listing its components (need a different word):

  • Data Objects refer to the Form system inside Specify that allows data to be viewed, added, edited and deleted.
  • Tasks refer to all other tools within Specify. These tools do not affect actual data, but allow users to manipulate the data in meaninful ways (Trees, Queries).
  • Preferences lists the modules in the actual Preferences module.

Each of these categories is further distinguished by user capabilities, or actions:

  • View allows a user to view data within the component.
  • Add allows a user to add data or create a new item.
  • Modify allows users to edit data within an item.
  • Delete allows users to delete data within the component.

 

Permissions and how it affects the Specify Application

When Specify starts up it checks the user permissions and loads various components accordingly. Therefore, Users that are not given permission to use modules in Specify will not see the corresponding buttons that connect users to that module. For instance, users that are not given permission to edit Trees will not have a Tree button on the Taskbar or have the ability to add the Tree button in the user Preferences tool.

Similarly, users that are not given permission to delete or add data in a module will not be offered the add or delete buttons.
