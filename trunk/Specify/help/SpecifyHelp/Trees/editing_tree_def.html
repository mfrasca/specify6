Tree Definition

The Tree Definition tool defines the ranks available in a Tree as well as with how these ranks are enforced in data entry and used in aggregating the full name. Tree Definition tools are available for each of the trees.

Click the Trees button on the Task Bar, then select a tree from the Sidebar list under Edit Tree Definition.

The resulting window displays a hierarchical list of included Ranks in a grid view.

A tree and form may not write to the database at the same time. Therefore, if a Tree or Tree Definition are open a form can be viewed but not edited. For example, if user A is editing the Taxon Tree Definition user B will not be allowed to edit a Taxon form.

Also, a Tree and Tree Definition can not be open at the same time. If the Tree Definition is open and the Tree is selected the Tree will replace the definition in the same tab, therefore closing the Tree Definition.

 

Change Tree Title

Click on the (edit) button next to the tree title to edit the title of the Tree. This is also the name given to the first node in te tree.

Note: If the save (or update) process is interrupted, the Tree will update, and a prompt will appear, the next time Specify is launched.

 

Predefined Levels Form

Edits to large Tree Definitions will require an ample amount of time to write to the data base. To aid this process, Specify has predefined ranks available for Taxon Trees. When adding a Taxon rank, the first dialog that pops-up will require a rank name and offer choices from the predefined ranks, as well as a choice to type in a custom rank name. Whenever possible, it is highly recommended that users choose a predefined rank.

There is a limit to the number of custom ranks available between each predefined rank. When custom ranks are no longer able to be added between the predefined ranks, the status bar message Unable to assign Rank will appear.

Note: Ranks must be defined in the Tree Definition before they can be entered as nodes in either a tree or form. For example, the City of Lawrence can not be added under Douglas County in a Geography Tree if the Tree Definition does not include a Rank for City.
