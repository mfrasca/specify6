Attachment Tools

Attachment tools offer the following functionality:

  1. view all attachments within a collection in an attachment browser
  2. add multiple attachments in a single action


Open the Attachment Tools by clicking the Attachment button on the task bar.

If Attachment Tools is not available on the task bar add it by choosing Specify > Preferences > Taskbar for Macintosh,
or Edit > Preferences > Taskbar for Windows and Linux,
then checking the Add Attachments checkbox.

Three functions appear on the sidebar:
  1. Show All Attachments
  2. Import Images
  3. Import Image Index

1. Show All Attachments

Click Show All Attachments in the sidebar to display a thumbnail view of all attachments for the current collection within the workspace. 

Note: The Attachment tools are limited to files in an 'Attachment' table, not files linked to records using a Linking Field display.

Attachment Levels
Attachments are stored at the same institutional level as their associated table and display dependent up on this level. For example, the Collecting Event table is at the Discipline Level and a pdf attached to a Collecting Event record will display for all collections within the discipline.

To Open an Attachment or Display a larger image:

  • Click an individual attachment thumbnail to display an image larger in a new tab in the workspace.
  • Click on non-image files to open the file in its native program (example: .exe opens the file in Excel).

Display more information about the attachment

Click the table icon next to the thumbnail in the workspace to open the attached record in a form view.
Click the  button next to the thumbnail in the workspace to display more information about the attached record or to open the attached record in a form view.
Click the  button located at the bottom of the workspace to display more information about the record in a separate 'information pane'. This pane also displays associated lat and long information on a map.
Click the  button located on the bottom of the workspace to display metadata associated with the attachment file.
Note: A useful example of metadata is the date, latitude and longitude information associated with a jpg photo when taken using a digital camera. The metadata information may be stripped from the file when the jpg image is loaded into an outside program such as a web-based image storage facility.
Note: Some file types such as png, bmp and gif contain little or no metadata information.

Export Attachment

Right-click on the thumbnail and click the Export Attachment button.


2. Import Attachments

The Import Attachments function allows multiple attachments to be associated to records by simply renaming attachments within a directory to match a unique field within the record. For example, if records within the Collection Object table include unique values for field number, simply rename attachment files to match the field number in the record.

To use this function click Import Attachments located on the Attachment side bar.

Choose which unique identifier to attach the files (as reflected by the name of the attachment).

Choose the directory/file to import.

Specify will display a message once the upload is complete.




If any files were not able to upload Specify will display additional information in a web browser window.

Import Image Mapping Files must first be created in a spreadsheet program and saved as either tab delimited or csv files.

The spreadsheet must contain 2 columns, one with the unique field value and one with the name of the attachment file. The unique field value and file names must match existing values and names exactly.
Note: more than one attachment can be associated with the same unique identifier.
Note: attachment files are not required to be named using the unique identifier.
