Introduction

Over 250 years of botanical and zoological exploration have produced billions of biological specimens in museums worldwide which document the distribution of life on earth. The Internet has changed the way we catalog, computerize and discover those specimens, but we are just at the beginning of utilizing the communication and computational power of the net. The goal of Specify 6 is to extend collections computing beyond simple specimen description, discovery and data retrieval--to capabilities for integration and synthesis that will enable researchers to utilize collections data for broader, quicker, research impact, in efficient, network-based collaborations.

Specify 6 is based on the same concepts as Specify 5, but has been completely rewritten in Java and can run on Windows, Mac OS X, and Linux. It is an open source software project with the source available at ***.

Before using Specify 6, it is important to understand its components and how they interact with each other.

 

Components in Specify

 

A. Menu Bar B. Taskbar C. Sidebar D. Workspace E. Trash Can F. Tab Bar / TabsG. Status Bar

 

Menu Bar

The Menu Bar across the top organizes commands and modules under menus and submenus.

 

Taskbar

The Taskbar located at the top of the Specify application is a set of buttons that launch a task-related module. The taskbar can be moved by clicking on the 2 vertical lines found on the left side of the taskbar and dragging it away from the application. To move the taskbar back to the application click the (close) button.

 

Sidebar

The Sidebar is a unique component that lists the actions (print, import) and data (types of forms) available within a module. The sidebar will change to reflect the tools available within the active module.

Resize the Sidebar by mousing over the line separating the Sidebar from the Workspace until a crossbar appears, then click and drag the line to the desired size.

 

Workspace

The Workspace is the area within the application that tasks are completed within. Many Workspace windows can be open at the same time, but only one can be active.

 

Trash Can

The Trash Can allows a saved item (Record Set, Query, Report or Information Request) listed on the Sidebar to be deleted by dragging and dropping it onto the image.

 

Tab Bar / Tabs

Tabs on the Tab Bar allow Workspace windows to be activated and closed. More than one window can be open at a time, but only one may be active in the workspace, these windows are shown as tabs on the tab bar under the workspace. Click a tab to activate the window.

The tab bar allows 11 tabs, but not all can be viewed at one time. When the tabs move off the tab bar, arrows ( ) become available for viewing and selecting tabs.

Tabs are labeled with an icon and module name. If two of the same modules are open the tab will reflect the number of modules that are open.

Status Bar

The Status Bar at the bottom of the application displays the User, Discipline and Collection name. When appropriate it also displays messages such as invalid data entry and the status of a search.

 

Levels within Specify

Specify 6 is designed to store all the data for an institution in one database. This may encompass many disciplines, and collections. Each collection within a discipline has unique data needs; therefore many of the tools within Specify, such as forms and trees, are also unique to each discipline, with individual data being unique to each collection. For this reason Specify opens to specific collections and permissions can be granted at both a collection and discipline level.

The following is a list of the levels within Specify and the modules that store data at that level.

 

Institution

  • Backup
  • Restore
  • Permissions

Division

Discipline

  • Schema Configuration (table formats, field formats, aggregation, field and table labels, pick lists, web links)
  • Agent table
  • Trees
  • Forms

Collection

  • Data Entry
  • Interactions
  • Query
  • Simple Search
  • Statistics
  • Record Sets

 

Change Collections

Each session of Specify opens to a collection. The first time Specify is opened a dialog will ask which collection the user wishes to open, then Specify will open to the last collection that was open.

To change a collection choose File > Change Collection, then choose a collection from the resulting dialog.

Available collections are based on the permissions granted to the current user.

Specify 6 displays the current collection name on the right side of the Status Bar.

Note: Selecting Change Collection closes the current collection as well as all open tabs.

 

Permissions -- some tools found in the documentation may not be available to all users in the permissions and not all users will see it! Check with your Specify administrator for your institution.
