Institution Levels

Specify mirrors the common levels of heirarchy within institutions.

  • Institution
  • Division
  • Discipline
  • Collection
  • Group
  • Person

After running the Setup Wizard the Divisions, Disciplines and Collections for each institution will need to be configured.

Tools for adding Divisions, Disciplines and Collections are found by clicking System > Collection Setup > Configuration. All tabs need to be closed before opening the Configuration.

Start adding to the Institution Hierarchy at the Division level and work your way down to Collection. Each is added at the level above it. For instance, Divisions are added to the Institution form.

Editing an existing Discipline must be executed on the Discipline form, as well as Division. A Collection can be edited either on the Discipline form or Collection form.

Add Divisions

Click the Institution button on the Side Bar to add Divisions.

At least one Division should appear in the Division sub form.

Click the (add) button to add a Division.

Name is the only required field and is used in any lists of Divisions within Specify as well as in the Institution Hierarchy.

Alt Name provides another field for name. This is often useful when a particular Division has changed names or uses a longer name than is applicable to the Hierarchy list.

URI stands for Uniform Resource Identifier and refers to a resource on the internet (a Division web site).

Is Accession Bound tells Specify that this Division does not share Accessions within the institution will have their own accession table and numbers.

 

Add Disciplines

Click the Division button on the Side Bar to add Disciplines.

The Division form includes the information entered when the Discipline was added. The new Discipline will be added in the Division that is currently displayed in the Workspace. Use the navigation control at the bottom of the window to step to the Division that will include the new Discipline.

In the Discipline sub form click the (add) button to add a Discipline.

A dialog listing all available Discipline types will display. Choose a discipline from the list. The chosen type will appear in the Type field. This name is used by the database in Specify.

Data Type indicates that the data base is for biological collections and is not editable.

Name allows the Discipline to be named by the Institution rather than by the provided Specify list. This name will display in any Discipline lists within Specify as well as on the Institution Hierarchy.

The Collections form displays under Disciplines on this form. It is to view Collection data only, Collections can not be added or edited from the Division form.

 

Add Collections

Collections are added in the Discipline form. Click the Discipline button on the Side Bar.

The Discipline information will appear at the top of the form with a Collections sub form underneath. The new Collection will be added in the Discipline that is currently displayed in the Workspace. Use the navigation control at the bottom of the window to step to the Discipline that will include the new Collection.

Click the (add) button to add a Collection.

A Numbering Scheme dialog will appear. This refers to the Catalog Number. Each Collection can either accept an existing numbering format for Catalog Numbers or create their own.

To accept a numbering scheme simply click on the correct one in the list (or use the down-arrow key).

Collection Form

The Collection Name will be used on any Specify Collection lists as well as in the Institution Hierarchy.

Add Collecting Event Automatically creates a new, unique Collecting Event record for each Collection Object. This is typically used in Botany.

ABCD Schema Fields is a subform for adding this information to the Collection.
